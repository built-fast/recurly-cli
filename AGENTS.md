# Recurly CLI Development Context

## Development Loop

Make changes, run `make check`, fix what it catches, repeat until green, then
push. `make check` runs fmt-check + vet + lint + tidy-check + check-surface +
test + test-e2e + vuln. Treat it as your inner-loop companion, not a final
hurdle.

## Repository Structure

```
recurly-cli/
├── cmd/                 # Command implementations (one file per resource group)
├── internal/
│   ├── client/          # Recurly API client wrapper, retry logic, error handling
│   ├── config/          # TOML configuration (api_key, region)
│   ├── output/          # Output formatting (table, JSON, jq filtering, fields)
│   ├── pagination/      # Generic paginator (Collect[T] with limit/all modes)
│   └── surface/         # CLI surface snapshot generator
│       └── cmd/gensurface/  # Executable that produces .surface
├── e2e/                 # BATS end-to-end tests (Prism mock server)
├── openapi/             # OpenAPI spec (api.yaml) — source for Prism mock
├── .github/workflows/   # CI pipeline (ci.yml)
├── main.go              # Entrypoint — delegates to cmd.Execute()
├── Makefile             # Build, lint, test, surface targets
├── .surface             # CLI surface snapshot (committed)
└── .golangci.yml        # Linter config (gocritic, paralleltest, cyclop)
```

## Recurly API Reference

Base URL: `https://v3.recurly.com` (US) or `https://v3.eu.recurly.com` (EU)

API version header: `Accept: application/vnd.recurly.v2021-02-25`

Key resource paths (see `openapi/api.yaml` for full spec — 130 endpoints):

- `/accounts` — CRUD, reactivate, deactivate, balance, notes
- `/accounts/{id}/billing_info` — billing info management
- `/accounts/{id}/coupon_redemptions` — coupon redemptions (list, create, remove)
- `/accounts/{id}/subscriptions` — account subscriptions
- `/accounts/{id}/invoices` — account invoices
- `/accounts/{id}/transactions` — account transactions
- `/subscriptions` — CRUD, cancel, reactivate, pause, resume
- `/invoices` — list, get, collect, void, mark-failed, line-items
- `/plans` — CRUD for subscription plans
- `/plans/{id}/add_ons` — plan add-on management
- `/items` — item catalog CRUD, activate, deactivate
- `/coupons` — CRUD, generate codes, restore, deactivate
- `/transactions` — list, get

## Testing

`make check` is the local CI gate. Run it before pushing.

```bash
make check             # All checks (local CI gate)
make test              # Go unit tests only
make lint              # golangci-lint
make vet               # go vet
make fmt               # Format code (gofmt)
make fmt-check         # Check formatting (fails if not formatted)
make tidy-check        # Verify go.mod/go.sum are tidy (non-mutating)
make test-e2e          # BATS e2e tests (requires Prism)
make build             # Build binary to ./recurly
make surface           # Regenerate .surface snapshot
make check-surface     # Verify .surface is up to date
make vuln              # govulncheck for dependency vulnerabilities
```

When iterating on a specific area, use targeted targets for faster feedback,
then finish with `make check` before pushing.

### Unit Tests

Unit tests live alongside source files as `*_test.go`. Each resource has a
corresponding `*_api.go` interface file to enable mocking. Tests use `t.Parallel()`
throughout (enforced by the `paralleltest` linter).

### E2E Tests

E2E tests use [BATS](https://github.com/bats-core/bats-core) (Bash Automated
Testing System) with a [Prism](https://github.com/stoplightio/prism) mock server
that validates requests against `openapi/api.yaml`.

The test runner (`e2e/run.sh`) handles the full lifecycle: build binary, start
Prism on a random port, run BATS tests, clean up. Test helpers live in
`e2e/test_helper.bash`.

Environment variables used by e2e tests:
- `PRISM_URL` — mock server base URL (set by run.sh)
- `RECURLY_BINARY` — path to compiled CLI binary (set by run.sh)

## Surface Snapshot

The `.surface` file at the repo root is a deterministic, sorted snapshot of
every command, argument, and flag in the CLI. It is generated by
`internal/surface/surface.go` and committed to version control.

**Purpose**: catch accidental command/flag changes in CI and provide a stable
reference for skill drift detection.

**Format** — one line per entry, sorted lexicographically:

```
CMD recurly accounts list
ARG recurly accounts get <account_id>
FLAG recurly accounts list --limit int
FLAG recurly --output string           # persistent flag on root
```

**Regenerate** after adding/removing/renaming commands, flags, or arguments:

```bash
make surface           # regenerate .surface
```

**CI enforcement**: `make check` includes `check-surface`, which fails with a
diff if `.surface` is stale. Always run `make surface` after command/flag changes
and commit the updated file.

## Skill System

`skills/recurly/SKILL.md` is an agent-facing reference document embedded into the
binary via `skills/embed.go`. It teaches AI agents how to use recurly-cli:
authentication, commands, flags, workflows, error codes, and decision trees.

### Commands

- `recurly skill` — prints SKILL.md to stdout (no auth required)
- `recurly skill install` — installs SKILL.md to the Claude Code skills directory

### Keeping SKILL.md in sync

Every command and flag mentioned in SKILL.md must match an entry in `.surface`.
After command/flag changes:

1. Run `make surface` to regenerate `.surface`
2. Update `skills/recurly/SKILL.md` if commands, flags, or workflows changed
3. Run `make check` — the `check-skill-drift` target will catch any mismatches

### Drift Detection

`scripts/check-skill-drift.sh` scans SKILL.md for `recurly <cmd>` and `--<flag>`
references and verifies each one exists in `.surface`. Known/accepted mismatches
can be baselined in `.surface-skill-drift`.

```bash
make check-skill-drift  # run drift check standalone
```

## Workflow: Command or Flag Changes

When you add, remove, or rename a command or flag:

1. Make the code change in `cmd/`
2. `make surface` — regenerate `.surface`
3. Update `skills/recurly/SKILL.md` if the change affects agent-visible behavior
4. `make check` — validates fmt, vet, lint, tests, e2e, surface, and tidy

## Tool Requirements

- **Go** 1.26+ (see `go.mod` for exact version)
- **golangci-lint** — multi-linter runner
- **govulncheck** — dependency vulnerability scanner
- **bats-core** — Bash testing framework (for e2e tests)
- **Node.js/npx** — required for Prism mock server
- **jq** — used by e2e test assertions

Install all dependencies via Homebrew:

```bash
brew bundle
```
